✅ STATUS: FIXED

================================================================================
  THE PROBLEM
================================================================================

The "Questões Pendentes" admin panel broke on load because of a response 
structure mismatch:

❌ Backend returns: { sucesso: true, dados: { questoes: [...], ... } }
❌ Frontend expected: response.questoes

Result: 100% failure rate when loading pending questions


================================================================================
  THE ROOT CAUSE
================================================================================

Line 290 of QuestoesPendentesTab.jsx had broken fallback logic:

  const response = await questoesService.listar(params);
  setQuestoes(response.dados?.questoes || response.questoes || []);
                                             ↑
                                   This path doesn't exist!

The backend's "dados" property exists, but "response.questoes" doesn't.
So the || fallback does nothing and questoes becomes an empty array.


================================================================================
  THE FIX
================================================================================

Changed the response extraction to properly handle the backend structure:

  const response = await questoesService.listar(params);
  
  let questoesData = [];
  if (response?.dados?.questoes) {
    questoesData = Array.isArray(response.dados.questoes) 
      ? response.dados.questoes 
      : [];
  } else if (response?.questoes) {
    questoesData = Array.isArray(response.questoes) 
      ? response.questoes 
      : [];
  }
  
  setQuestoes(questoesData);
