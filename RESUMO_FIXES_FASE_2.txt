❓ FAQ

Q: Why did the panel break?
A: Frontend wasn't validating API response structure properly, and when API
   returned unexpected format, it would crash trying to render null items.

Q: Why don't approved questions appear?
A: Two reasons combined:
   1. Frontend used wrong param 'limit' instead of 'limite'
   2. Backend ignored explicit status filters for admins

Q: Do I need to change database?
A: No. All changes are code-only. Database schema remains the same.

Q: Will this break existing data?
A: No. Existing questions keep their current status. This just fixes the
   display and filtering logic.

Q: Can collaborators see rejected questions?
A: Only if they're assigned specific permissions (current code shows them
   their own questions regardless of status). Admin can see all statuses.
