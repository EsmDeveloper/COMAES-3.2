🔐 PROTEÇÕES IMPLEMENTADAS:

┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│ 1. LOCK DE BANCO DE DADOS (SERIALIZABLE)                                  │
│    └─ Previne race condition em participacao-ativa                       │
│    └─ Transação ACID garante atomicidade                                │
│    └─ Sem risco de múltiplas inscrições simultâneas                     │
│                                                                             │
│ 2. VALIDAÇÃO DE TIPO DE TORNEIO                                          │
│    └─ Backend verifica tipo_torneio ∈ ['generico', 'especifico']       │
│    └─ Se específico: valida disciplina_competida == disciplina_especif  │
│    └─ Erro 400 se incompatível                                          │
│                                                                             │
│ 3. VERIFICAÇÃO DE EXPIRAÇÃO                                              │
│    └─ Dupla validação: na hora e ao tentar inscrever                   │
│    └─ Impossível burlar com timestamps antigos                          │
│    └─ Rankings congelados (imutáveis) ao expirar                       │
│                                                                             │
│ 4. PARTICIPAÇÃO SIMULTÂNEA                                               │
│    └─ Query: ParticipanteTorneio.findOne({                             │
│    │           usuario_id, status='confirmado',                        │
│    │           posicao_congelada=false                                │
│    │         })                                                        │
│    └─ Se encontrado: erro 409 (Conflict)                              │
│    └─ Lock de BD durante verificação                                 │
│                                                                             │
│ 5. FILTRAGEM DE DISCIPLINAS                                              │
│    └─ Backend verifica blocos existentes por disciplina                 │
│    └─ Não retorna disciplinas sem conteúdo                             │
│    └─ Evita interface vazia/confusa para usuário                       │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

🎮 FLUXO DO USUÁRIO - ANTES vs DEPOIS

┌─ ANTES ───────────────────────────────────────────────────────────────────┐
│                                                                           │
│ 1. Usuário entra em EntrarTorneio.jsx                                   │
│ 2. Interface mostra 3 disciplinas (sempre)                              │
│ 3. Pode clicar em qualquer uma                                         │
│ 4. Se clicou em outra via: pode estar em 2+ torneios simultaneamente   │
│ 5. Sem verificação de tipo de torneio                                  │
│ 6. Torneios expirados continuam aceitando inscrições                   │
│                                                                           │
│ ❌ Problemas:                                                            │
│    - Usuário confuso (3 opções mesmo sem blocos)                      │
│    - Múltiplos torneios = confusão de estado                          │
│    - Sem proteção de expiração                                         │
│                                                                           │
└───────────────────────────────────────────────────────────────────────────┘

┌─ DEPOIS ──────────────────────────────────────────────────────────────────┐
│                                                                           │
│ 1. Usuário entra em EntrarTorneio.jsx                                   │
│ 2. Frontend busca /api/torneios/ativo/disciplinas                      │
│ 3. Interface mostra APENAS disciplinas com blocos                      │
│    - Se genérico: mostra 1-3 (com blocos)                             │
│    - Se específico: mostra 0-1 (apenas aquela)                        │
│ 4. Usuario clica em disciplina                                         │
│ 5. Frontend verifica participacao-ativa                                │
│    - Se tem ativa: mostra erro + bloqueado                           │
│    - Se sem ativa: permite continuar                                 │
│ 6. Backend valida tudo novamente                                       │
│    - Tipo de torneio                                                 │
│    - Expiração                                                        │
│    - Participação simultânea                                         │
│ 7. Se OK: inscrição confirmada                                        │
│                                                                           │
│ ✅ Benefícios:                                                           │
│    - Interface limpa (sem opções vazias)                              │
│    - Proteção contra múltiplos torneios                              │
│    - Expiração automática funciona                                   │
│    - Dupla validação (frontend + backend)                            │

🐛 POSSÍVEIS PROBLEMAS E SOLUÇÕES:

Problema 1: Endpoint /api/torneios/ativo/disciplinas retorna []
Causa: Nenhum bloco publicado naquela disciplina
Solução: Admin deve criar e publicar blocos no gerenciador

Problema 2: User continua vendo "Torneio Ativo" mesmo após expiração
Causa: Frontend em cache ou intervalo de refresh muito grande
Solução: F5 ou aguardar próxima chamada automática (hard refresh)

Problema 3: Erro "ParticipanteTorneio not included" em join
Causa: Include não foi configurado corretamente no endpoint
Solução: Verificar associations.js — já devem estar set up

Problema 4: Participacao-ativa retorna true mesmo após congelamento
Causa: Query não filtra posicao_congelada corretamente
Solução: Verificar WHERE em verificarParticipacaoAtiva

Problema 5: Disciplina de torneio específico não aparece
Causa: Não há blocos publicados para essa disciplina
Solução: Admin associar blocos e publicá-los
