fix(database): add contexto column to blocos_questoes table

This commit resolves the critical issue where all /api/blocos endpoints 
were returning HTTP 500 errors due to a database schema mismatch.

PROBLEM:
--------
- The BlocoQuestoes Sequelize model defined a 'contexto' field
- The blocos_questoes database table did NOT have this column
- When creating/reading blocks, Sequelize tried to access a non-existent column
- MySQL returned: "Unknown column 'contexto' in field list"
- Express caught this as a 500 Internal Server Error
- Users were completely blocked from creating or viewing blocks

SOLUTION:
---------
1. Executed SQL migration to add ENUM('torneio', 'teste') column
2. Set default value to 'torneio' for backward compatibility
3. Made column nullable to handle existing data
4. Verified Sequelize can access the column
